# IMPACKTA AI — Chatbot Server

Standalone Express.js RAG chatbot server for the IMPACKTA AI website. Answers visitor questions using a curated knowledge base embedded in LanceDB, with Gemini 2.5 Flash generating the responses.

---

## Stack

| Layer | Technology |
|-------|-----------|
| Runtime | Node.js 18+ |
| Language | TypeScript |
| Server | Express.js |
| Embeddings | `@xenova/transformers` — local, free, no API |
| Vector DB | LanceDB (embedded, no separate process) |
| LLM | Google Gemini 2.5 Flash |

---

## Environment Variables

Create a `.env` file in the `server/` directory. Use `.env.example` as the template:

```bash
cp .env.example .env
```

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `GEMINI_API_KEY` | Yes | Google Gemini API key | `AIza...` |
| `PORT` | No | Port the server listens on. Defaults to `3001` | `3001` |
| `FRONTEND_URL` | Yes | Frontend origin allowed by CORS. Must match exactly | `https://impackta.online` |
| `TRANSFORMERS_CACHE` | No | Override HuggingFace model cache directory. Useful in containers | `./.cache` |

> **Getting a Gemini API key:** Go to [aistudio.google.com](https://aistudio.google.com), sign in with a Google account, and create an API key.

> **FRONTEND_URL in development:** Set to `http://localhost:3000` when running the Next.js frontend locally.

## First-Time Setup

### 1. Install dependencies

```bash
cd server
npm install
```

> On first install, `@xenova/transformers` will download the embedding model (~80 MB) on the first run. This is cached automatically.

### 2. Set up environment variables

```bash
cp .env.example .env
# Edit .env and fill in GEMINI_API_KEY and FRONTEND_URL
```

### 3. Build the vector database (run once)

This reads `data/knowledge.md`, splits it into chunks, embeds each chunk locally, and stores everything in LanceDB.

```bash
npm run ingest
```

Expected output:
```
Reading knowledge document...
Split into 13 chunks
Connecting to LanceDB...
Embedding chunks...
  [13/13] chunk_12...
All chunks embedded
Writing to LanceDB...
Done. 13 chunks stored in LanceDB at .../data/lancedb
```

> Re-run `npm run ingest` any time `data/knowledge.md` is updated.

## Running the Server

### Development (with auto-reload)

```bash
npm run dev
```

Server starts at `http://localhost:3001` (or your `PORT`).

### Production

```bash
npm run build
npm start
```

## API

### `POST /chat`

No authentication required.

**Request:**

```http
POST /chat
Content-Type: application/json

{
  "message": "What services does IMPACKTA AI offer?"
}
```

**Response 200:**

```json
{
  "answer": "IMPACKTA AI offers three service tiers..."
}
```

**Response 400 — missing message:**

```json
{ "error": "message is required" }
```

**Response 400 — message over 1000 characters:**

```json
{ "error": "message too long" }
```

**Response 500:**

```json
{ "error": "Something went wrong" }
```

### Quick test with curl

```bash
curl -X POST http://localhost:3001/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "What does IMPACKTA AI do?"}'
```

## Updating the Knowledge Base

All chatbot knowledge lives in `data/knowledge.md`. To update what the bot knows:

1. Edit `data/knowledge.md`
2. Re-run the ingest script:
   ```bash
   npm run ingest
   ```
3. Restart the server

## Deployment Notes

- This server is deployed **separately** from the Next.js frontend.
- Set `FRONTEND_URL` to the exact production frontend domain (e.g. `https://impackta.online`).
- The `data/lancedb/` directory is **not committed to git**. You must run `npm run ingest` on the server after each deployment.
- If running in a Docker container or serverless environment, set `TRANSFORMERS_CACHE` to a writable path so the embedding model can be cached between runs.
- The embedding model (~80 MB) is downloaded on first use. In production, pre-warm by running `npm run ingest` before taking traffic.
- Recommended: run behind a reverse proxy (nginx / Caddy) with HTTPS.
